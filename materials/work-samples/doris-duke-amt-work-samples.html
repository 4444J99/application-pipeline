System Architecture — Real-Time Audience-Participatory Performance Engine
Omni-Dromenon Engine architecture. Audience members (left) control continuous parameters via phone-based sliders. Inputs flow to the Core Engine, which aggregates them through a three-axis weighted consensus algorithm — spatial proximity to the stage, temporal recency, and cluster agreement — before applying performer override resolution. The Performance SDK (top right) provides React interfaces for both audiences and performers. The Audio Synthesis Bridge (middle right) routes consensus values to SuperCollider/Max/MSP via OSC and browser-native WebAudio. Genre presets (bottom right) encode each form's aesthetic priorities as weighting ratios. Designed for 1,000+ concurrent participants at <2ms P95 latency and 20 consensus updates/second. Open-source (MIT).

Four Working Examples — Genre-Specific Performance Applications
Four genre-specific examples ship with the engine. Each demonstrates a different audience-performer dynamic and genre preset. Generative Music routes audience consensus to SuperCollider synthesis via OSC, prioritizing temporal responsiveness. Generative Visual maps collective input to WebGL shader uniforms, prioritizing spatial position within the installation. Choreographic Interface integrates camera-based pose detection with consensus-weighted movement mapping — audience proximity to the dancer determines influence. Theatre Dialogue uses a branching narrative engine where audience consensus selects story paths while performers lock critical beats. The grant funds deploying these in live venue conditions with performing artists across dance, theatre, and music — testing and refining the genre presets based on performer feedback.

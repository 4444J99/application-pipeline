# File Naming Convention for Application Materials

## Format

```
Anthony-Padavano-[Target-Job-Title]-Resume.pdf
Anthony-Padavano-[Target-Job-Title]-Cover-Letter.pdf
```

## Rules

1. **Full professional name first** — `Anthony-Padavano` (or `Anthony-James-Padavano` if needed for disambiguation)
2. **Target job title second** — matches the role title, abbreviated if needed. Max 5-6 words.
3. **Document type last** — `Resume` or `Cover-Letter`
4. **Dashes between words** — not underscores, not spaces
5. **PDF only** — unless posting specifies otherwise
6. **No dates, versions, or company names in filename** — ATS strips these anyway and they look amateur
7. **Resume and cover letter naming must match** — same convention on both

## Examples

```
Anthony-Padavano-Forward-Deployed-Engineer-Resume.pdf
Anthony-Padavano-Forward-Deployed-Engineer-Cover-Letter.pdf

Anthony-Padavano-Staff-Full-Stack-Engineer-Resume.pdf
Anthony-Padavano-Staff-Full-Stack-Engineer-Cover-Letter.pdf

Anthony-Padavano-Solutions-Engineer-Resume.pdf
Anthony-Padavano-Solutions-Engineer-Cover-Letter.pdf
```

## Experience Title — Never "Independent Engineer"

The resume's experience section title should match the target role context, not announce self-employment:

| Target Role | Resume Title (instead of "Independent Engineer") |
|-------------|------------------------------------------------|
| SWE / Platform | Software Engineer & Platform Architect |
| Technical Writer / Docs | Documentation Architect & Technical Writer |
| DevRel / Advocate | Developer Advocate & Technical Educator |
| FDE / Solutions | Systems Architect & Technical Deployment Lead |
| Agent / AI | AI Systems Architect & Agent Framework Builder |
| Staff / Senior | Staff Engineer & Systems Architect |

**Why:** "Independent" triggers the contractor/freelancer pattern. "Self-employed" triggers the founder penalty (43% fewer callbacks per research). Lead with the function, not the employment structure.
